byfile モード

byfile モードでは、ファクトリ定義ファイル で定義されたファクトリを使用して値が生成されます。コマンドの形式は次の通りです：

randog byfile FACTORY_PATH [...] [--regenerate PROB_REGEN] [--discard PROB_DISCARD] [--csv ROW_NUM] [--error-on-factory-stopped] [common-options]

引数 FACTORY_PATH は factory の外部定義ファイル のファイル名です。このファイルは python プログラムであり、以下のように factory を作成して変数 FACTORY にバインドするプログラムである必要があります。

import uuid

FACTORY = randog.factory.from_example({
    "uuid": uuid.uuid4,
    "name": "",
    "age": 20,
})

# (optional) Settings used for CSV output
CSV_COLUMNS = ["uuid", "name", "age"]

注釈

factory定義ファイルでは、import randog を省略できます。

引数とオプション

  • FACTORY_PATH [...]:

    • 1つ以上の factory定義ファイル のパス

  • --regenerate PROB_REGEN (default=0.0):

    • factory が生成した値をそのまま使用せずに再生成する確率。このオプションは factory が完全にランダムでない値を生成する場合に効果がある。

  • --discard PROB_DISCARD (default=0.0):

    • factory が生成した値をそのまま使用せずに廃棄する確率。廃棄が生じた場合、生成される数は --repeat/-r や --list/-L や --csv で指定した数より少なくなる。

  • --csv ROW_NUM (optional):

    • 指定すると、ROW_NUM 個の生成されたオブジェクトをCSV形式で出力します。このオプションを使用する場合、辞書を生成する factory を使用し、定義ファイル内にカラムを定義する CSV_COLUMNS を定義することが推奨されます。

  • --error-on-factory-stopped (optional):

    • 指定した場合、factory が StopIteration によって値を生成できなくなった際にエラーを発するようになります。指定しない場合、先述のケースで単に生成を停止します。

  • common-options

    • 共通オプション

実行例

最も簡素な例は次の通りです：

randog byfile factory_def.py

定義ファイルでデータベースのレコード1つに相当するdictを生成する factory を定義している場合、下記のように複数個生成することでテスト用データをそろえることができます。

# Generate list which contains 10 values
randog byfile factory_def.py -L 10

1つ1つを別々のファイルに出力しながら複数個生成したいことがあるかもしれません。その場合は、以下のように -O や -r を活用します。

# Repeat 10 times and output each of them into out_001.json, out_002.json, ... with json format
randog byfile factory_def.py -r 10 -O 'out_{:03}.json' --json

欠番ありのタイムスタンプをPKにしている場合などは、生成した値の一部を破棄したい場合があるかもしれません。その場合、 --discard や --regenerate を使用することで出力を欠損させることができます。例：

# output at most 20 values (each value will be discarded at 10% probability)
randog byfile factory_def.py --repeat 20 --discard 0.1

# output exactly 20 values (each value will be regenerated at 10% probability)
randog byfile factory_def.py --repeat 20 --regenerate 0.1

CSV として出力

CSV形式で出力したい場合は、--csv オプションを使用します。各フィールドの値は、定義ファイル内で定義された CSV_COLUMNS によって決定されます。

# output CSV which contains 20 rows
randog byfile factory_def.py --csv 20

警告

factoryが辞書以外のオブジェクトを生成する場合や定義ファイルに CSV_COLUMNS が定義されていない場合でも --csv オプションを指定すれば CSV 形式で何かしら出力しますが、推奨されません。将来的に、その場合の動作を変更する可能性もあります。

CSV出力においても、--repeat/-r オプションと --output/-O オプションで複数ファイルに出力することができます。

# output 10 CSV files; each file contains 20 rows
randog byfile factory_def.py --csv 20 -r 10 -O 'out_{:03}.csv'

このページ上部の例では CSV_COLUMNS を文字列のリストとして定義していましたが、辞書のキーを指定する文字列の代わりにフィールドを返す関数を指定することもできます。次の例では、3番目のフィールドとして、age の値を加工した文字列を使用します。

import uuid

FACTORY = randog.factory.from_example({
    "uuid": uuid.uuid4,
    "name": "",
    "age": 20,
})

# output example: 17642547-0a4c-4897-a8da-2d495558b8fa,d40s8Jqs,20 years old
CSV_COLUMNS = [
    "uuid",
    "name",
    lambda d: f"{d['age']} years old",
]

欠番ありのタイムスタンプをPKにしている場合などは、生成した値の一部を破棄したい場合があるかもしれません。その場合、 --discard や --regenerate を使用することで出力を欠損させることができます。例：

import uuid
from datetime import datetime, timedelta
import randog

def timestamp_iter():
    next = datetime(2002, 1, 1, 0)
    while True:
        yield next
        next += timedelta(hours=1)

FACTORY = randog.factory.randdict(
    timestamp=randog.factory.by_iterator(timestamp_iter()),
    name=randog.factory.randstr(),
    age=randog.factory.randint(0, 100),
)

CSV_COLUMNS = ["timestamp", "name", "age"]

# output at most 20 rows (each row will be discarded at 10% probability)
randog byfile factory_def.py --csv 20 --discard 0.1

# output exactly 20 rows (Gaps of 'timestamp' at 10% probability)
randog byfile factory_def.py --csv 20 --regenerate 0.1

注釈

--discard で行を欠落させると、出力される行は --csv で指定した数より少なくなります。

注釈

--regenerate で行を再生成させると、内部で値が生成される回数は --csv で指定した数より多くなります。

環境変数による動作パターンの変更

factory定義の細かい設定を環境変数によって変えられるようにすることも、実用的なアイデアの一つです。例えば、以下の定義ファイルでは id の初期値を環境変数で指定できます。

import itertools
import os
import randog

initial_id = int(
    os.environ.get("INIT_ID", "0")
)

FACTORY = randog.factory.randdict(
    id=randog.factory.by_iterator(itertools.count(initial_id)),
    name=randog.factory.randstr(),
    age=randog.factory.randint(0, 100),
)

環境変数の指定には、シェルの標準的な方法のほか、randogのenvオプションを使用することができます。以下の例はいずれも同じように動作します。

# Can use it in bash, etc., but not in powershell
INIT_ID=5 randog byfile factory_def.py

# Can use it in any shell
randog byfile factory_def.py --env INIT_ID=5

注釈

次のように、複数の環境変数を指定することもできます。

randog byfile factory_def.py --env INIT_ID=5 VAR=foo
randog byfile factory_def.py --env INIT_ID=5 --env VAR=foo

注釈

定義ファイルをインポート可能にしたい場合、環境変数の読み込みを if __name__ == "__randog__" で実施した方がよいかもしれません。詳細は インポート可能な定義ファイル をご覧ください。
