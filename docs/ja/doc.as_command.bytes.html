bytes モード

byte モードでは、バイト列を生成します。コマンドの形式は次の通りです：

randog bytes [--length LENGTH] [--fmt FORMAT] [common-options]

警告

このバイト列生成では、暗号学的に安全であることは保証されません。認証用トークンの生成など、セキュリティ要件があるケースでは、secrets モジュール等の使用を検討してください。

引数とオプション

  • --length LENGTH (optional):

    • 生成されるバイト列の長さ。--length 5 のように整数値を指定することや、--length 3:8 のように範囲を指定することができる。

  • --fmt FORMAT (optional):

    • 指定した場合、指定したフォーマットで出力する。フォーマットは 独自の書式コード で指定する。

  • common-options

    • 共通オプション

実行例

最も簡素な例は次の通りです：

randog bytes --output ./result.dat

次の例のように、バイト列長を指定することができます。

# generate bytes of length 16
randog bytes --length 16 --output ./result.dat

# generate bytes of length 8 - 16
randog bytes --length 8:16 --output ./result.dat

注釈

後述する 出力フォーマット を使用しない場合、バイナリデータのまま出力されます。コンソール画面に出力しようとすると windows powershell などでは正常に出力されないため、上述の例のように --output オプションを使用してファイル出力します。

警告

このバイト列生成では、暗号学的に安全であることは保証されません。認証用トークンの生成など、セキュリティ要件があるケースでは、secrets モジュール等の使用を検討してください。

出力フォーマット

デフォルトではバイナリデータが出力されますが、次の例のように 書式コード を使用することで出力フォーマットを変更できます。

# generates a value casted by str()
randog bytes --fmt s
# generates a value expressed in hexadecimal with padding zeros on the left.
randog bytes --fmt '0>20x'

オプション --base64 を使用することで、base64 にエンコードすることもできます。--fmt を併用した場合、base64 エンコードの結果を文字列とみなして --fmt に指定したフォーマットで出力します。

# output base64-encoded value such as 'QIj/'
randog bytes --base64

# output base64-encoded value such as '  QIj/'
randog bytes --base64 --fmt '>6s'
# output base64-encoded value such as '"QIj/"'
randog bytes --base64 --json

繰り返し生成

-r オプションを使用することで、複数回値を生成することができます。

ただし、出力フォーマットを指定しない場合はバイナリで出力され、複数の値は区切り文字無しで連続で出力されます。これを避けるためには、次の例のように、--output オプション を使用して生成ごとに異なるファイルへ出力します。

# Repeat 10 times; results are output as binary continuously without a delimiter
randog bytes -r 10

# Repeat 10 times; results are separated into different files
randog bytes -r 10 --output 'result_{0}'

書式コード

bytes オブジェクトには書式コードが定義されていませんが、このモードでは Standard Format Specifier に近い書式コードを使用できます.

# generates a value casted by str()
randog bytes --fmt s
# generates a value expressed in hexadecimal with padding zeros on the left.
randog bytes --fmt '0>20x'
# generates a value expressed in binary notation with 4 digits each divided by '_'.
randog bytes --fmt _b
# generates a value expressed in hexadecimal with prefix '0x'.
randog bytes --fmt '#x'

利用可能な表現型は以下です:

  • s: 文字列。出力は str() の出力と同一です。例: b'ab\xff'.

  • c: 文字列。出力は str() の出力の内側部分と同一です。例: ab\xff.

  • b: 2進数。出力される数値は2を基数とします。

  • x: 16進数。出力される数値は16を基数とします。10進で9を超える数字には小文字が使われます。

  • X: 16進数。出力される数値は16を基数とします。10進で9を超える数字には大文字が使われます。'#' が指定された場合に付く接頭辞の '0x' も大文字の '0X' になります。
