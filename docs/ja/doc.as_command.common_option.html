共通オプション

コマンド実行のいくつかのオプションは全てのモードで使用することができます。

出力形式の変更

通常、生成されたオブジェクトはそのまま print() 関数で出力されますが、以下のオプションを使用することで出力形式を変更することができます。

  • --repr: 生成されたオブジェクトを repr() 関数で変換して出力します。

  • --json: 生成されたオブジェクトをJSON形式で出力します。標準的なJSON形式が定められていないオブジェクトは str() 関数で文字列に変換してからJSON化されます。

例:

python -m randog time --json
python -m randog time --repr

ファイル出力

以下の例のように --output オプションを使用することで、結果をファイル出力することができます。

python -m randog time --output ./out.txt

# output to out.txt in UTF-16 LE with line-separator '\r\n'
python -m randog byfile ./factory_def.py -O out.txt -X utf_16_le --O-ls CRLF

# {now} will be replaced by the current time
python -m randog time --output './out_{now:%Y%m%d%H%M%S}.txt'

上の例のように、オプション --output-encoding (-X) や --output-linesep (--O-ls) を使用することで、出力時の エンコード と改行文字を指定できます。

上の例で {now} を使用しているように、下記のプレースホルダーを使用できます (書式指定子 も使用できます)。

  • {0} (int):

    通し番号。--repeat オプションを使う場合や byfile モードで複数の定義ファイルを順番に使用する場合、連番になる。一度しか使用しないなら {} でもよい。

  • {def_file} (str):

    byfile モードで使用している定義ファイル名。ただし、末尾の .py は除去される。byfile モード以外では空文字列に置き換えられる。

  • {repeat_count} (int):

    --repeat オプションで繰り返す際の連番。--repeat を使用しない場合は 0 に置き換えられる。

  • {factory_count} (int):

    byfile モードで複数の定義ファイルを使用する際に定義ファイルごとに付けられる連番。byfile モード以外の場合は 0 に置き換えられる。

  • {now} (datetime.datetime):

    現在日時。

  • (環境変数)

    {HOME} のように環境変数をプレースホルダーとして使用できる。

繰り返し

繰り返し値を生成するオプションには、 --list (-L) と --repeat (-r) があります。

繰り返し生成した値を1つのリストにまとめて出力したい場合は、以下のように --list オプションを使用します。

# generate ONE list which contains 3 objects; each element conforms to factory_def.py.
python -m randog byfile factory_def.py --list 3

一方で、繰り返し生成した値を1つずつ個別に出力したい場合、以下のように --repeat オプションを使用します。

# generate and output 3 times
python -m randog byfile factory_def.py --repeat 3

注釈

--repeat を使用して生成した値を別々のファイルへ1つずつ出力したい場合、以下のようにプレースホルダーとともに --output オプションを使用します。

# output to 'out_1.txt', 'out_2.txt', and 'out_3.txt'
python -m randog factory_def.py --repeat 3 --output './out_{}.txt'

# output to 'out_0001.txt', 'out_0002.txt', and 'out_0003.txt'
python -m randog factory_def.py --repeat 3 --output './out_{:04}.txt'

プレースホルダーのルールは 標準的な python のフォーマット と同一です。

使用できるプレースホルダーについては ファイル出力 を参照してください。

シード値

通常はランダムに値を生成しますが、出力結果をコントロールしたい場合、--seed を使ってシード値を指定します。以下の例のようにシード値が同じであれば同じ結果を返します。

# first
python -m randog str --seed 42

# second; the result is the same as the first
python -m randog str --seed 42

警告

シード値が同一でも、python のバージョンが変わると生成される値が変わる場合があります。詳しくは 再現性についてのドキュメント をご覧ください。

また、randog や依存パッケージのバージョンアップによっても生成する値が変わる可能性があります。

シード値を指定しない場合はランダムなシード値が使用されます。使用されるシード値は ログ出力される ため、以下のようにしてシード値を確認することができます。

# generate str with log
python -m randog str --log-stderr DEBUG

確認したシード値をメモしておけば、次回、そのシード値を使用することで生成を再現できます。

環境変数を変更

とくに byfile モードで、定義ファイルに値を受け渡す目的で環境変数を指定したいかもしれません。bash 等では VAR=VAL python -m randog ... のように1行で環境変数を指定できますが、powershellなど一部のシェルではそれができません。

そこで、randog では環境変数を指定するオプションを用意しています。次のように --env を使用することで環境変数を指定できます。

python -m randog byfile factory_def.py --env CHARSET=0123456789abcdef

このような実行は、例えば以下のような定義ファイルを使用する際に有用です。

import os
import randog.factory

FACTORY = randog.factory.randstr(
    length=4,
    # Get the value specified for charset from an environment variable
    charset=os.environ["CHARSET"],
)

ログ

デフォルトでは randog が出力するものも含めてすべてのログが無視されます (例外は後述) が、ログ出力するように設定することができます。

警告

これは実験的な機能です。将来、削除もしくは大幅な修正をする可能性があります。

ログ出力について、以下のオプションのいずれか1つを使用できます。

  • --log-stderr <LEVEL>:

    指定したレベル以上のログを標準エラー出力へ出力します。デフォルトではトレースバックを省略するよう設定されますが、--log-stderr ERROR-full のようにレベルの後ろに "-full" をつけると、例外が発生した場合などのトレースバックも出力されます。

  • --log <LOGGING_CONFIG_PATH>:

    指定したログ設定ファイルを使用します。ファイルは JSON もしくは YAML 形式で、configuration dictionary schema に従う必要があります。--log-stderr とは異なり、トレースバックは省略されません。

警告

YAML 形式の設定ファイルを使用する場合、PyYAML をあらかじめインストールしておく必要があります。

設定ファイルを書くにあたって、randog が使用するロガーの情報が必要かもしれません。その場合は、ロギングについてコマンド実行に限定せず記載している ログ出力 をご覧ください。

注釈

警告 はログとは別の仕組みで設定します。警告 を参照してください。

注釈

実は、コマンド実行中のエラーメッセージも logging を使用しています。ログ設定ファイルに disable_existing_loggers: true を指定することで、コマンド実行中のエラーメッセージ出力設定を上書きできます。(標準ライブラリの仕様では disable_existing_loggers のデフォルト値が true ですが、randog のコマンド実行時の標準エラー出力設定については disable_existing_loggers を明示的に true にした場合にのみ上書きします。)

警告

つまり、disable_existing_loggers: true を指定すると、設定次第では異常終了時のエラーメッセージが表示されなくなるということです。

警告

デフォルトで、警告 は標準エラー出力に出力されますが、設定を変えることができます。

警告

これは実験的な機能です。将来、削除もしくは大幅な修正をする可能性があります。

randog の警告は --quiet/-q オプションで非表示にできます。すべての警告を非表示にしたい場合は python の -W オプションを使用してください (こちら を参照)。

注釈

randogの警告のみを -W で非表示にすることも仕様上はできるはずですが、問題があるようです。randog の --quiet/-q を使用してください。

ちなみに、randog のコマンド実行が出す警告のカテゴリは randog.RandogCmdWarning です。
