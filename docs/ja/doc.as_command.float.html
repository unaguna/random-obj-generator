float モード

float モードでは、浮動小数点数を生成します。コマンドの形式は次の通りです：

python -m randog float [MINIMUM MAXIMUM] [--p-inf PROB_P_INF] [--n-inf PROB_N_INF] [--nan PROB_NAN] [--exp-uniform] [--fmt FORMAT] [common-options]

引数とオプション

  • MINIMUM (optional):

    • 最小値。指定がない場合の動作は、randfloat の仕様に準ずる。

  • MAXIMUM (optional):

    • 最大値。指定がない場合の動作は、randfloat の仕様に準ずる。

  • --p-inf PROB_P_INF (optional, default=0.0):

    • 正の無限大を生成する確率。

  • --n-inf PROB_N_INF (optional, default=0.0):

    • 負の無限大を生成する確率。

  • --nan PROB_NAN (optional, default=0.0):

    • NaNを生成する確率。

  • --exp-uniform (optional):

    • 指定した場合、桁数 (2を底とする対数) が一様になる。

  • --fmt FORMAT (optional):

    • 書式指定ミニ言語 で記載された出力フォーマット

  • common-options

    • 共通オプション

実行例

最も簡素な例は次の通りです。0から1の間の数値を出力します。

python -m randog float

次の例のように、生成する値の範囲を指定することができます：

# generates a value between 0.01 and 10^100
python -m randog float 0.01 1e+100

注釈

decimal モード も使用できます。

小数部がない整数を生成したい場合は int モード を使用してください。

Infinity and NaN

オプション引数を指定することで無限大やNaNを生成の候補に含めることができます。

# Generates positive infinity with 10% probability
python -m randog float 0 1e+100 --p-inf 0.1

# Generates NaN with 15% probability
python -m randog float --nan 0.15

無限大やNaNの表現は python の標準 (inf, -inf, nan) を使用します。出力を他の言語で書かれたプログラムで読み込む場合などで支障があれば、--json オプションを使用してJSON形式 (Infinity, -Infinity, NaN) で出力することもできます。

# Generates Infinity, -Infinity, or, NaN
python -m randog float --p-inf 0.4 --n-inf 0.4 --nan 0.2 --json

出力フォーマット (有効桁数等)

以下のように出力フォーマットを 書式指定ミニ言語 で指定することができます。

# output numeric such as '4.20e+98', which has (2+1) significant digits
python -m randog float 0.01 1e+100 --fmt .2e

注釈

小数部の長さは --fmt .2f のように指定できますが、小数部の長さが決まった数値を生成する場合は decimal モード が向いています。

繰り返し生成

たいてい、1つ生成しただけでは足りないので、次のようにして複数回出力することになるでしょう。

# Repeat 10 times
python -m randog float -r 10

# Generate list which contains 10 values
python -m randog float -L 10

確率分布 (桁数の分布を一様に)

デフォルトでは生成の確率分布は一様分布です。たとえば、0.1 から 1000.0 の値を生成する場合、0.1 が生成される確率、5.0 が生成される確率、500.0 が生成される確率はすべて同一です。

特にこだわらない場合は一様分布でも問題ありませんが、様々な桁数の値が生成されてほしい場合はこれでは問題があります。上述の例だと3桁の数値 (100.0 - 1000.0) が生成される確率は 90% であり、1未満の数が生成される確率はわずか 0.1% しかありません。つまり、生成される値の大半が3桁なのです。

桁数の分布を均一にするためには --exp-uniform を使用します。このオプションを使用することで桁数が小さい数に大きい比重がかけられ、桁数はおおむね均一になります。より正確には、2進数表記した際の桁数 floor(log2(x)) の分布が一様になります。ただし、0の桁数は0とし、正の数と負の数は別々に確率を持ちます。たとえば、-8 から 24 までの整数を生成する場合、以下の事象はすべて同じ確率です。

  • -8.0 超 -4.0 以下

  • -0.25 超 0.125 以下

  • 0.0

  • 1.0

  • 0.125 以上 0.25 未満

  • 8.0 以上 16.0 未満

なお、当該桁数の数の一部しか生成範囲に入っていない場合、その桁数になる確率は減少します。上述の例では、16-32 のうち半分しか生成範囲に含まれないため、他の桁数に比べて確率は半分になります。

注釈

浮動小数点数は 2 の -1022 乗の小ささまで表すことができるため、生成範囲に 0 を含めると、絶対値が小さい値ばかりになる傾向にあります。たとえば、範囲を 0 から 100 にした場合、桁数は -1022 から 7 までなので、生成値の90%は 2 の -100 乗より小さい値になります。

これを望まない場合、decimal モード を使用してください。確率分布が10進表記での桁数に基づくようにはなりますが、オプション --decimal-len で最小単位を制限できます。
