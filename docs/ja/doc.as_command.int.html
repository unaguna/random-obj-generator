int モード

int モードでは、整数値を生成します。コマンドの形式は次の通りです：

python -m randog int MINIMUM MAXIMUM [--exp-uniform] [--fmt FORMAT] [common-options]

引数とオプション

  • MINIMUM:

    • 最小値

  • MAXIMUM:

    • 最大値

  • --exp-uniform (optional):

    • 指定した場合、桁数 (2を底とする対数) が一様になる。

  • --fmt FORMAT (optional):

    • 書式指定ミニ言語 で記載された出力フォーマット

  • common-options

    • 共通オプション

実行例

最も簡素な例は次の通りです。指定した区間の値をランダムに1つ出力します。

python -m randog int 0 100

注釈

小数値を生成したい場合は、decimal モード や float モード を使用してください。

出力フォーマット (0-パッディング等)

以下のように出力フォーマットを 書式指定ミニ言語 で指定することができます。

# output 0-padded integer such as '00000042'
python -m randog int 0 100 --fmt 08

# output 0-padded integer such as '+42'
python -m randog int 0 100 --fmt +

繰り返し生成

たいてい、1つ生成しただけでは足りないので、次のようにして複数回出力することになるでしょう。

# Repeat 10 times
python -m randog int -r 10

# Generate list which contains 10 values
python -m randog int -L 10

確率分布 (桁数の分布を一様に)

デフォルトでは生成の確率分布は一様分布です。たとえば、1 から 10000 の値を生成する場合、1 が生成される確率、50 が生成される確率、10000 が生成される確率はすべて同一です。

特にこだわらない場合は一様分布でも問題ありませんが、様々な桁数の値が生成されてほしい場合はこれでは問題があります。上述の例だと4桁の数値 (1000 - 9999) が生成される確率は 90% であり、1桁の数が生成される確率はわずか 0.1% しかありません。つまり、生成される値の大半が4桁なのです。

桁数の分布を均一にするためには --exp-uniform を使用します。このオプションを使用することで桁数が小さい数に大きい比重がかけられ、桁数はおおむね均一になります。より正確には、2進数表記した際の桁数 floor(log2(x)) の分布が一様になります。ただし、0の桁数は0とし、正の数と負の数は別々に確率を持ちます。たとえば、-15 から 47 までの整数を生成する場合、以下の事象はすべて同じ確率です。

  • -15 以上 -8 以下

  • 0

  • 1

  • 8 以上 15 以下

  • 16 以上 31 以下

なお、当該桁数の数の一部しか生成範囲に入っていない場合、その桁数になる確率は減少します。上述の例では、32-63 のうち半分しか生成範囲に含まれないため、他の桁数に比べて確率は半分になります。

注釈

あまり大きな違いは生じませんが、10進表記の桁数に基づいて確率を決定したい場合、decimal モード を使用できます。その際、 --decimal-len=0 を使用することで生成値を整数に限定できます。
