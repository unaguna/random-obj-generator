timedelta モード

timedelta モードでは、経過時間を生成します。コマンドの形式は次の通りです：

python -m randog timedelta [MINIMUM MAXIMUM] [--unit UNIT] [--iso | --fmt FORMAT] [common-options]

引数とオプション

  • MINIMUM (optional):

    • 最小値を 簡易書式 で指定する。指定がない場合の動作は、randtimedelta の仕様に準ずる。

  • MAXIMUM (optional):

    • 最大値を 簡易書式 で指定する。指定がない場合の動作は、randtimedelta の仕様に準ずる。

  • --unit UNIT (optional):

    • 生成される値の最小単位を 簡易書式 で指定する。指定がない場合の動作は、randtimedelta の仕様に準ずる。

  • --iso (optional):

    • 指定した場合、ISO-8601 形式 で出力する。

  • --fmt FORMAT (optional):

    • 指定した場合、指定したフォーマットで出力する。フォーマットは 独自の書式コード で指定する。

  • common-options

    • 共通オプション

実行例

最も簡素な例は次の通りです。timedelta の値を出力します。

python -m randog timedelta

次の例のように、生成する値の範囲を指定することができます：

# generates a value between 1 day and 7 day
python -m randog timedelta 1d 7d

生成される値の最小単位を指定しない場合、適当な長さに調整されます。多くの場合は次のように手動で指定する必要でしょう。

# generates a value between 0 day and 1 day with minimum unit 1 second
python -m randog timedelta 0 1d --unit 1s

デフォルトでは 簡易書式 で出力されますが、次のようにオプションを指定することで出力形式を 独自の書式コード で指定したフォーマットや ISO-8601 形式 に変更できます：

# generates a value with ISO-8601 format
python -m randog timedelta 0 1d --unit 1s --iso

# generates a value with the specified format　(x days xx:xx:xx)
python -m randog timedelta 0 7d --unit 1s --fmt '%D days %H:%M:%S'
# generates a value with the specified format　(x:xx:xx)
python -m randog timedelta 0 7d --unit 1s --fmt '%tH:%M:%S'

たいてい、1つ生成しただけでは足りないので、次のようにして複数回出力することになるでしょう。

# Repeat 10 times
python -m randog timedelta -r 10

# Generate list which contains 10 values
python -m randog timedelta -L 10 --json --iso

簡易書式

このモードでは、経過時間を 1h30m や 1d2h3m4s5ms6us のような独自のフォーマットで表現します。

このフォーマットは、以下の要素を1つ以上連結することで経過時間を表現します。

ディレクティブ

意味

Xd

X は日数です。例えば、2d は "2日間" を意味します。

Xh

X は時間 (hour) です。例えば、2h は "2時間" を意味します。

Xm

X は分 (minute) です。例えば、2m は "2分間" を意味します。

Xs

X は秒数です。例えば、2s は "2秒間" を意味します。

Xms

X はミリ秒数です。例えば、2ms は "2ミリ秒間" を意味します。

Xus

X はマイクロ秒数です。例えば、2us は "2マイクロ秒間" を意味します。

0
ゼロ。0d, 0h, 0s 等と同じ経過時間を表します。
ゼロの場合は単位が意味を成さないため、単位なしの表記も認められています。

注釈

月単位や年単位で区間を指定することはできません。python の timedelta では日より大きい単位は管理しておらず、たとえば1か月間と30日間を区別できないためです。

書式コード

pythonのtimedeltaには書式コードが定義されていませんが、このモードでは独自の書式コードを使用することができます。

ディレクティブ

意味

例

%D

timedelta.days に等しい

0, 1, 2, ...

%H

時間 (hour) 部分 (0埋めして2桁表記)

00, 01, ..., 23

%tH

時間 (hour) 単位の総時間 (小数点以下切り捨て)

0, 1, 2, ...

%M

分 (minute) 部分 (0埋めして2桁表記)

00, 01, ..., 59

%tM

分単位の総時間 (小数点以下切り捨て)

0, 1, 2, ...

%S

秒部分 (0埋めして2桁表記)

00, 01, ..., 59

%tS

秒単位の総時間 (小数点以下切り捨て)

0, 1, 2, ...

%f

秒数の小数部分 (0埋めして6桁表記)

000000, 000001, ..., 999999

典型的には、以下のようなフォーマットが使用されます：

  • %D %H:%M:%S

    • "140時間" は "5 20:00:00" と表現されます

  • %tH:%M:%S

    • "140時間" は "140:00:00" と表現されます

注釈

月単位や年単位で区間を指定することはできません。python の timedelta では日より大きい単位は管理しておらず、たとえば1か月間と30日間を区別できないためです。
