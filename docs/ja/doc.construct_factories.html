Factory の生成

randog ではランダムに値を生成するために factory オブジェクトを使用します。生成のルールは factory を作る際に指定します。

もし型以外の生成条件にこだわらないのであれば、単に値の例を from_example に渡すだけで factory を作ることができます。他の条件も指定したい場合は、生成する値の型に応じた factory のコンストラクタを使用してください。

>>> import randog.factory

>>> # create a factory simply
>>> factory_a = randog.factory.from_example("")
>>> generated_a = factory_a.next()
>>> assert isinstance(generated_a, str)

>>> # create a factory with conditions in detail
>>> factory_b = randog.factory.randstr(length=16)
>>> generated_b = factory_b.next()
>>> assert isinstance(generated_b, str)
>>> assert len(generated_b) == 16

Nullable

None も生成されることがある factory を作りたい場合は or_none メソッドを使用します。

>>> import randog.factory

>>> factory = randog.factory.from_example("")
>>> factory_nullable = factory.or_none(0.1)

>>> # a string
>>> generated_a = factory.next()
>>> # a string or None
>>> generated_b = factory_nullable.next()

注釈

常に None を返す factory を作りたい場合は const を使用してください。

注釈

通常、or_null factory が値を生成する際は、まず factory を使用するかNoneを返すかをランダムに決定し、factory の値を使用する場合にのみ値を生成させます。ただし、or_none(..., lazy_choice=True) のように引数を指定した場合、その or_null factory が値を生成する際は、まず factory で値を生成した後で、それを採用するかNoneを返すかをランダムに決定します。

この違いは、非ランダムな factory を使用する場合等に影響します。

合成型 (Union type)

複数の型のうちのいずれかの値をランダムに生成する factory をいくつかの方法で作ることができます。

注釈

factory を nullable にしたい場合、つまり None との合成型にしたい場合、代わりに or_none を使用してください。

from_example を使用する場合、引数に Example を使用してください。以下の例では -1, "", and True を例として使用しているため、factory は整数値、文字列、真偽値のいずれかを生成します。

>>> from randog import Example
>>> import randog.factory

>>> factory = randog.factory.from_example(Example(-1, "", True))

>>> for _ in range(10):
...     generated = factory.next()
...     assert isinstance(generated, (int, str, bool))

候補となる factory を用意する場合、union を使用します。以下の例の factory は、値の生成のたびに、2つの factory (randint と randbool) からランダムに選び、それが生成した値を結果として返します。

>>> import randog.factory

>>> factory = randog.factory.union(
...     randog.factory.randint(0, 10),  # integer
...     randog.factory.randbool(),  # True or False
... )

>>> for _ in range(10):
...     generated = factory.next()
...     assert isinstance(generated, (int, bool))

注釈

通常、union factory が値を生成する際は、まずどの factory を使用するかをランダムに決定し、その factory にのみ値を生成させます。ただし、union(..., lazy_choice=True) のように引数を指定した場合、その union factory が値を生成する際は、まずすべての factory で値を生成した後でそのうちどの値を採用するかをランダムに決定します。

この違いは、非ランダムな factory を使用する場合等に影響します。

ランダムな選択 (Randomly choice)

指定した値のうちのいずれかをランダムに返す factory を作る場合は randchoice を使用します。

>>> import randog.factory

>>> factory = randog.factory.randchoice("allow", "deny")

>>> for _ in range(10):
...     generated = factory.next()
...     assert generated in ["allow", "deny"]

注釈

特定の列挙型の値をランダムに生成する factory を作る場合は randenum を使用します。こちらもご参照ください： Enum factory

定数

常に同じ値を返す factory を作る場合は const を使用します。

>>> import randog.factory

>>> # same as `factory = randog.factory.randchoice("python")`
>>> factory = randog.factory.const("python")

>>> for _ in range(10):
...     generated = factory.next()
...     assert generated == "python"

結果の加工

factory の生成値に対する加工をあらかじめ定義することができます。この加工で生成値の型を変更することもできます。

>>> import randog

>>> # use post_process to format the random decimal value
>>> factory = (
...     randog.factory.randdecimal(0, 50000, decimal_len=2)
...                   .post_process(lambda x: f"${x:,}")
... )

>>> # examples: '$12,345.67', '$3,153.21', '$12.90', etc.
>>> generated = factory.next()
>>> assert isinstance(generated, str)
>>> assert generated[0] == "$"

生成される値が dict であってその要素に処理をしたい場合、以下の例のように post_process の代わりに post_process_items を使うことで簡単に書けます。

>>> import randog

>>> # use post_process_items to format the random decimal value '["count"]'
>>> factory = (
...     randog.factory.randdict(
...         name=randog.factory.randstr(),
...         count=randog.factory.randdecimal(0, 50000, decimal_len=2),
...     ).post_process_items(count=lambda x: f"${x:,}")
... )

>>> # examples: {'name': 'sir1w94s', 'count': '$12,345.67'}, etc.
>>> generated = factory.next()
>>> assert isinstance(generated["count"], str)
>>> assert generated["count"][0] == "$"

注釈

dict でない値が生成された場合は post_process_items による変換がスキップされるため、or_none や union と組み合わせて使うこともできます。

カスタム Factory

randog が提供していない型の値も、randog の文脈でランダムに生成することができます。そのためには、関数やイテレータ (ジェネレータイテレータ も含む) やカスタム factory を使用します。通常、わざわざ randog を通さずとも関数やイテレータをそのまま使えばよいのですが、dict や list の要素を生成するためにはこの方法が必要になります。

>>> import itertools
>>> import random
>>> import uuid
>>> import randog.factory

>>> # define custom factory
>>> class MailAddressFactory(randog.factory.Factory[str]):
...     def _next(self):
...         return random.randint(1, 10) * "a" + "@example.com"

>>> factory = randog.factory.from_example({
...     # use iterator (https://docs.python.org/3/library/itertools.html#itertools.count)
...     "id": itertools.count(1),
...     # use function
...     "uuid": uuid.uuid4,
...     # use function
...     "name": lambda: random.randint(1, 10) * "a",
...     # use custom factory
...     "mail": MailAddressFactory(),
... })
>>> generated = factory.next()

>>> assert isinstance(generated, dict)
>>> assert generated["id"] == 1
>>> assert isinstance(generated["uuid"], uuid.UUID)
>>> assert isinstance(generated["name"], str)
>>> assert set(generated["name"]) == {"a"}
>>> assert isinstance(generated["mail"], str)
>>> assert generated["mail"].endswith("@example.com")

注釈

factory のコンストラクタである by_callable, by_iterator を使用して factory を作ることもできます。

警告

有限のイテレータを使用することもできますが、一度そのイテレータが使い切られるとその factory は値を生成することができなくなります。
